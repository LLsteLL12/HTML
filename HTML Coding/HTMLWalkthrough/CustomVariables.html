Box model, Margin and Padding.

CSS Box Model

CSS box models construct themselves on 4 major parts of coding. Those being :-
Content - This is the contents of the box be it text or images.
Padding - This is a transparent border that creates a space around the contents.
Border - A border that goes around the contents and the padding.
Margin - The margin clears an area outside of the border that is transparent.

{ margin: "" ; outside the border padding: "" ; inside the border }

For the two, there are shorthand ways to vary the amount of each side. (top, bottom, left and right). These methods save you from copying -
margin-top:0px;
margin-right:0px;
margin-bottom:20px;
margin-left:2px;
1) Margin/padding: 10px; - This applies 10px to all 4 sides of the element. 2) Margin/padding: 10px 20px; - This applies "10px" to the top and bottom. Applies "20px" to the left and right. 3) Margin/padding: 10px 5px 20px 5px; - This applies "10px" to the top, "5px" to the right, "20px" to the bottom and "5px" to the right.
The "value" can also be percentages instead of pixels. The application of the values go clockwise.

The difference is that with the values being in pixels, this will give a set number for the edit to happen on. Whereas with percentages, this takes into account the users device dimensions of the screen and adjusts the values accordingly.

Display.

The "display" attribute specifies the behavious of tags that are paired with it. This allows us to alter the visual state of what is being used from its default "block" which would use all of the available space on a line even if it isn't necessary.

We can alter the "display" of a tag or an element into 3 alternatives.
1) Inline - This displays an element or a tag with the space that it requires and no more, allowing other text to use up the remaining space on a line. Removing any affect of the height and width values.
2) Block - This displays the tag or element as a default block that will start on a new line and spare no space on said line.
3) Inline-Block - This gives us the freedom to alter the width and height desired to place the tag or element on an inline level and in forms of blocks.

Z-Index is a stacking order property we can put on an item to give it a particular order to show under or over other items on a page. Media Queries

Media queries are used as a rule to apply different styles to different media types and devices. Media types also aid us in delovering a more responsive designs accross a spectrum of device types.

They can be used to check properties such as :-
- The height and width of a viewport.
- The width and height of a device.
- The orientation of a device (potrait or landscape)
- The resolution of a device screen.
Media queries can be used in code to change how a page looks on different devices dependent on the screen size to ensure proper and fluent use of said page.

The way we would create a media query is to use the following code :-

@media(max-width:480px) and (min-width:320px) {
body{
background-color: pink; }

.parent-container{
display:flex;
justify-content:space-evenly;
flex-direction: row;
flex-wrap: wrap;
background-color: black; }

This code would be for the device parameters of a mobile phone or "small screen" as the "max-width" and "min-width" are set as the general pixel sizes of a mobile screen. These can be altered to register a different screen size and maybe have a different style to fit the new perameters. For instance, on a large screen (1025px - 1200px); a webpage may have a fully accessible menu accross the top of the screen to direct you too a certain page. However due to a mobile device having sucha reduced pixel size (320px - 480px); you might consider putting the menu into a hamburger menu state instead for ease of access to the user.

nth Child Pseudo Selectors

The ":nth-child(n) selector" matches ever element that is the nth child (regardless of the parent).
It has 3 uses and those are:-
- n = 1 2 3 4 5 6 7
- 3n = 3 6 9 12 15 18
- 3n-2 = 1 4 7 10 13 16 19

n = a number, keyword (odd or even), or a formula such as "3n nth term". The code would be written using an < html element, id, class>:nth-child(< number,keyword or formula>) { "other code"}
Items inside of the " { } " will affect the targeted items.

For instance. If we had a webpage with 3 sets of paragraphs on it then used the following:-
#IDname :nth-child(2) { color: gold; }

Then the 2nd paragraph text would be changed to the colour gold as the number after ":nth-child" being "(2)" it allocates the change to the second paragraph. If the item "(2)" was changed too "(odd)", then paragraphs thats are odd numbers (1, 3, 5, 7) would be chnaged to gold. Vice versa with "(even)".

If we were to use "3n", then it would reciprocate the maths equivalent of "3n" and alter every 3rd sequenced item. "n" being the multiple used to identify a particular sequenced item. "3n-2" would use a similar equivalent apart from it would take 2 from each item. Instead of starting at the third item, it would start the changes from the first. If we were to change the "2" to a different number, then that would be the step change to the sequence of alterations. For example: If instead of "3n-2" , we had "3n-1" then it would start the sequential changes from the second item, then the fourth, then the sixth.

Box and Text Shadows

These elements are self explanatory in terms of giving a shadow to a form of container using "box" (header, footer, div, article etc.) and a main body of text using "text" (heading, h1/2/3... paragraph etc ). The rules for each are identicle apart from the "box shadow" has the ability to have an inset shadow.

In order for them to work, they require 2 values: - h-offset = The horizontal offset of the shadow Positive values = right, negative values = left. - v-offset = The verticle offset of the shadow. Positive values = below, negative values = above.

There are additional values we can add such as : - color - For example, using red, hex or rgba value. This is required if you are using Safari - blur - The blur radius. The higher the numbered value, the more blurred the shadow will appear. - spread - The spread radius is the size of the shadow. (positive values increase the size of the shadow. Whereas negative values decrease the size of the shadow). - inset - This changes the shadow from an outer appearing shadow (outset), to an inner appearing shadow (inset)

The way we would write this into code woukd be as follows:
Box-shadow element{ box-shadow: h-offset v-offset / blur, spread, color, inset; (These will be in pixel values taken in that specific order.) }

For example:- div{ box-shadow: 50px -50px 10px 20px rgba(160, 9, 9, 0.5) inset; }

Text Shadows

The text-shadow property adds a shadow specifically to text.
The values we would use would be the same as for a "box shadow" except it would be specified for a text element..

For example:-
h1{ text-shadow: 50px 50px 10px 20px rgba(103, 9, 9, 0.5) inset; }
